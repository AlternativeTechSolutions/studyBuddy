# Study Buddy — Local MVP (Gemini edition)

A minimal local app to test the core idea: paste notes or upload a PDF → get a study guide + flashcards generated by Google's **Gemini API (free tier)**, with the output streaming live into the UI as it generates. No local model install required, no cost on the free tier. No auth, no billing, no database — just proving the generation pipeline is worth building on.

## Setup

1. **Get a free Gemini API key** at https://aistudio.google.com/apikey — sign in with a Google account, no credit card required.

2. **Install Node.js** (v18+) if you don't have it: https://nodejs.org

3. **Install project dependencies:**
   ```bash
   npm install
   ```

4. **Set up your env file:**
   ```bash
   cp .env.example .env
   ```
   Open `.env` and paste your key in place of `your-api-key-here`.

5. **Run it:**
   ```bash
   npm start
   ```

6. Open **http://localhost:3000** in your browser.

## What it does

- Paste text, or upload a `.pdf`/`.txt` file
- Click "Generate Study Materials"
- Watch the raw model output stream live into the "Live model output" panel as it generates
- Once done, get a formatted study guide + 8–15 flashcards in a flip-card review UI
- Every generated set is saved as a JSON file in `/data` so it survives restarts — check that folder to see token usage per run

## Cost and limits

**Free**, within Google's free-tier limits: currently 1,500 requests/day on Gemini 2.5 Flash, with a 1 million token/minute cap — generous enough for solo development and testing. If you hit a `429` error, you've hit the rate limit; wait a bit and retry (the app surfaces this as a clear error message rather than crashing).

**Privacy note:** on the free tier, Google's terms allow your prompts and outputs to be used to improve their models. Fine for MVP testing with non-sensitive content; if that matters later, moving to a paid API key or Vertex AI removes that data usage.

Limits and pricing can change — check https://ai.google.dev/gemini-api/docs/pricing for the current numbers before relying on this for anything beyond testing.

## What's intentionally NOT here yet

- No PPTX or audiobook generation (validate the text pipeline first — those are the expensive/complex additions)
- No user accounts, no Stripe, no multi-user support
- No spaced-repetition scheduling (flashcards just cycle in order)
- Minimal markdown rendering (headings/bold/lists only — good enough to read a study guide, not a full parser)

## Switching models or providers later

The generation logic lives entirely in `streamStudyMaterials()` in `server.js`. To try a different Gemini model, just change `GEMINI_MODEL` in `.env` (e.g. `gemini-2.5-flash-lite` for higher throughput/lower quality, or a paid Pro model once you're off the free tier). To switch providers entirely, that function is the only thing that needs rewriting — everything else (upload, storage, streaming UI, flashcard review) stays the same.

## Next steps once this proves out

1. If generation quality and speed feel right on the free tier → decide if it's viable long-term or if you'll need a paid key at scale
2. If quality needs work → tighten the prompt in `server.js`, or try Gemini 2.5 Pro if you have paid access
3. Add PPTX generation as the next increment, then audio last (it's the most expensive per-unit)
